## üìù Pull Request Guidelines
We don't have strict rules, but the following format is appreciated:

### Title Format
Please use a prefix to describe the change:
*   `[Feat]` : New features
*   `[Fix]` : Bug fixes
*   `[Docs]` : Documentation changes
*   `[Style]` : UI/Design adjustments (CSS, Layout)
*   `[Refactor]` : Code restructuring without behavior changes

**Example:** `[Docs] Add installation steps for Arch Linux`

### Description
*   **What** is changing?
*   **Why** is this change necessary?
